# Markdown 文件阅读器 — 设计文档

**日期：** 2026-05-11  
**状态：** 待审核

## 1. 概述

一个本地桌面应用，用于阅读 Markdown (.md) 文件，界面风格参考 GitHub 的渲染效果。用户可以拖拽单个 .md 文件打开，也可以指定文件夹进行浏览。

**核心目标：** 替代用 VS Code 查看 README.md 等文件的场景，提供 GitHub 级别的阅读体验，尤其是代码块的语法高亮和复制按钮。

## 2. 架构

```
┌──────────────────────────────────────┐
│          pywebview 桌面窗口           │
│  ┌────────────────────────────────┐  │
│  │      前端 (HTML/CSS/JS)        │  │
│  │  - marked.js  解析 Markdown    │  │
│  │  - highlight.js  代码高亮      │  │
│  │  - GitHub 风格 CSS             │  │
│  │  - 复制按钮 / 标题锚点链接      │  │
│  └──────────┬─────────────────────┘  │
│             │  JS Bridge (pywebview)  │
│  ┌──────────▼─────────────────────┐  │
│  │      Python 后端 (逻辑层)       │  │
│  │  - 读取本地 .md 文件            │  │
│  │  - 遍历文件夹，列出 .md 文件     │  │
│  │  - 处理拖拽（解析文件路径）       │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

**数据流：**
1. 用户操作（拖入文件 / 打开文件夹 / 点击文件列表）→ Python 接收
2. Python 读取文件内容 / 扫描目录 → 通过 JS Bridge 传给前端
3. 前端 marked.js 解析 Markdown → highlight.js 高亮代码 → 渲染 HTML
4. 用户交互（点击复制按钮、点击锚点链接）→ 纯前端处理

## 3. 前端渲染

### Markdown 解析：marked.js
- 轻量快速，支持 GFM 扩展（表格、任务列表、删除线），后续可零成本启用
- 本地引入，不依赖网络

### 代码高亮：highlight.js
- 自动检测语言，覆盖主流编程语言
- 使用 `github` 主题，与 GitHub 观感一致
- 代码块右上角复制按钮：
  - 默认显示复制图标
  - 点击后短暂显示"已复制 ✓"（2 秒后恢复）
  - 使用 Clipboard API 实现

### 标题锚点链接
- 鼠标悬停标题时，左侧出现 `#` 锚点图标
- 点击后更新页面 URL hash，支持页面内定位
- 行为与 GitHub 一致

### 整体样式
- 参考 GitHub 排版：字体系统、行高 1.6、合理边距
- 内容区最大宽度 900px，居中显示
- 表格、引用块（左边框）、代码块样式均参考 GitHub

## 4. 文件操作

### 拖拽打开
- 初始状态显示拖拽区域，提示"拖入 .md 文件或点击打开文件夹"
- 仅接受 `.md` 后缀文件，其他后缀提示不支持
- 拖入文件后直接进入阅读模式（无侧边栏）

### 文件夹浏览
- 菜单"文件 → 打开文件夹"弹出系统文件夹选择对话框
- 左侧侧边栏列出所有 `.md` 文件（按文件名排序）
- 点击列表项切换内容，当前文件高亮
- 侧边栏宽度可拖拽调整（200px ~ 400px）
- 侧边栏可通过菜单"视图 → 切换侧边栏"折叠/展开

### 菜单栏
- **文件：** 打开文件 (Ctrl+O)、打开文件夹 (Ctrl+Shift+O)、退出
- **视图：** 切换侧边栏

## 7. 打包为 .exe

使用 PyInstaller 打包为单个可执行文件：

```bash
pyinstaller --onefile --windowed --add-data "frontend;frontend" main.py
```

- `--onefile`：打包为单个 .exe
- `--windowed`：不显示命令行窗口
- `--add-data`：将前端文件嵌入 .exe

打包后在 `dist/` 目录生成 `main.exe`，可独立运行，无需安装 Python 环境。

## 8. 范围边界

### 包含
- 拖拽 .md 文件打开
- 文件夹浏览（侧边栏文件列表）
- GitHub 风格渲染（marked.js + highlight.js）
- 代码块复制按钮
- 标题锚点链接
- 打包为 .exe

### 不包含

- 实时编辑 / 热更新
- Markdown 编辑功能
- 多标签页
- 主题切换
- 搜索功能
